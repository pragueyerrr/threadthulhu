## Frontend Stack

```
frontend/
├── index.html              — dark background, fullscreen canvas
├── src/
│   ├── main.jsx            — React entry point
│   └── App.jsx             — Pixi.js city renderer (all logic lives here)
└── public/
    ├── points.json         — 206k tweet positions (generated by script 06)
    ├── districts.json      — district metadata + centroids
    └── tweets.json         — full tweet text, lazy-loaded on click
```

**Why Pixi.js v7 + pixi-viewport:**
- pixi-viewport v5 has peer dependency conflicts with pixi v7 — use `--legacy-peer-deps`
- Pixi v8 exists but pixi-viewport compatibility is uncertain — stay on v7 for now

---

## Script 06 — Export Frontend Data

Added `pipeline/06_export_frontend_data.py` which splits `full_mapped.json` into
three browser-friendly files:

| File | Size | Contents |
|------|------|----------|
| points.json | ~15MB | x, y, cluster_top, cluster_sub, likes, id — everything to draw the city |
| districts.json | tiny | district metadata, keywords, centroids (normalised 0-1) |
| tweets.json | ~50MB | full tweet text indexed by id — lazy-loaded on first click |

Coordinates normalised to 0–1 range in the export. Frontend multiplies by `WORLD_SIZE=8000`.

## Bugs Fixed This Session

| Bug | Cause | Fix |
|-----|-------|-----|
| `RangeError: Maximum call stack size exceeded` | `Math.min(...206k items)` blows JS stack | Replaced with manual loop |
| City had too much empty space | `viewport.fit()` fitted world not data | Computed actual data bounding box + 5% padding, fit to that |
| Labels only visible when very zoomed in | Fade threshold too high (0.06) | Lowered to 0.03 so labels appear at default city view |
| Tweet panel off-screen when zoomed in | Positioned bottom-right | Moved to top-right, always visible |
| URLs in tweet text not clickable | Plain text render | Split on URL regex, wrap in `<a>` tags |

---

## What the Frontend Does Now

- Renders 206k coloured dots on a dark canvas, one colour per top-level district
- Zoom + pan with mouse wheel / drag / trackpad (pixi-viewport)
- Landmark tweets (high likes) rendered slightly larger
- District labels appear on zoom, scale to consistent on-screen size
- Click any dot → nearest tweet loads in a panel (top-right)
  - Tweet text with clickable URLs
  - Date + like count
  - "View on X →" link (opens original tweet)
  - tweets.json lazy-loaded on first click (~50MB, cached after)

## District Naming Session

Reviewed all 28 top-level districts. Key decisions:

**Merged into The Agora:** D00, D20, D26, D27 — all were clusters of @reply
conversations directed at specific friend groups rather than thematic content.
Naming them after people felt exclusionary. The Agora (public square) captures
the conversational nature.

**D05 note:** "wife" appeared as a keyword but the district is genuinely about
animals — Visa tweets about animals with/to his wife. Named The Animal Kingdom,
not the wife keyword.

**D06 note:** "visa · visakanv" district is a threading artifact (he starts
self-reply threads with @visakanv). Renamed Deep Dive District — that's what
it actually represents.

## Decision: Tooltips for District Descriptions

District names like "The Marketplace", "The Interior", "The Forum" are evocative
but not immediately obvious to someone who doesn't know the archive. Decision:
**hover tooltips** — one sentence description appears under the label on hover.
Keeps the map clean; meaning is one hover away.

Next session: finalise the 24 descriptions, then implement tooltips in the frontend.
